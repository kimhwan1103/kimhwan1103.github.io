# 리스트

## 1. 리스트의 정의와 표기법

- 정의 : 길이가 n(음이 아닌 정수)인 리스트는 n개의 원소들을 순서대로 모아 놓은 컬렉션을 뜻함
- 표기 많은 수학자들은 이를 n-튜플이라고도 부르면 괄호 안에 원소들을 쉼표로 구분하여 ($z_1$, ..., $z_n$ )형태로 표기함
- 두 리스트가 같은 조건 : 두 리스트가 완전히 같으려면 길이가 똑같아야 하고 같은 순서로 동일한 원소들을 가지고 있어야 함

## 2. 리스트의 중요한 규칙들

- 반드시 유한한 길이여야 함 : 리스트는 정의상 무조건 유한한 길이(음수가 아니여함)를 가져야함 따라서 무한히 이어지는 것은 리스트가 아님
- 길이가 0인 리스트 : 길이가 0인 리스트는 빈 괄호인 ()로 표기합니다. 굳이 길이가 0인 대상을 리스트로 인정하는 이유는 앞으로 정리할 선형대수학의 여러 정리들이 사소한 예외 없이 성립하도록 만들기 위해서임

## 3. 집합(Set)과의 결정적 차이점

리스트는 일반적인 유한 집합과 비교했을 때 순서와 반복이라는 두 가지 측면에서 확연히 다름

- 순서가 중요 : 집합은 원소의 순서가 무의미해서 집합 {3, 5}와 {5, 3}이 완전히 같음. 하지만 리스트는 순서가 의미를 가지므로 리스트 (3,5)와 (5,3)은 서로 다른 리스트임
- 반복이 의미를 가짐 : 집합은 중복된 원소를 무시하기 떄문에 {4, 4}나 {4,4,4}나 모두 원소가 4 하나뿐인 집합 {4}와 같음. 반면 리스트에서는 원소가 반복되는 것이 의미가 있음. 즉 리스트 (4,4)와 (4,4,4)는 애초에 길이가 다르기 때문에 서로 다른 리스트로 취급됨

## 4. 리스트와 벡터의 관계

### 4-1. 수학적 & 선형대수학적 관점

- 개념적 차이 : 선형 대수학에서 벡터는 벡터 공간의 원소라는 추상적인 개념임. 이 추상적인 대상에 계산을 할 수 있도록 수치화 한것이 바로 스칼라 리스트임.
- 1:1 대응(좌표계) : 길이가 n인 실수 리스트 $(z_1, z_2, \dots, z_n)$ 은 $\mathbb{R}^n$ 공간에 존재하는 하나의 벡터를 나타냄. 리스트는 앞서 정리한 대로 순서가 절대적이므로, 리스트의 첫 번째 원소는 x축 두번째 원소는 y축 등 벡터의 각 차원과 정확히 매핑 될수 있음

![리스트와 좌표의 순서](/assets/img/posts/linear-algebra/list/order-geometry.png)

### 4-2. 컴퓨터 공학 및 AI 관점

- 자료구조로서의 Vector : 컴퓨터 공학에서 말하는 Vector는 리스트라는 추상적인 수학 개념을 메모리상에서 연속된 동적 배열로 구체화한것임.
- AI 모델에서의 활용 (Embedding & Tensor) : 고도화된 인공지능 시스템(LLM, GNN, etc)에서 텍스트 토큰이나 그래프의 노드가 가지는 의미적 특징은 수학적 공간의 벡터로 변환됨. 그리고 이 벡터는 실제 하드웨어 메모리상에서 수백~수천개의 길이를 가지는 실수형 리스트로 저장되어 GPU에서 행렬 연산을 거치게 됨. 즉 리스트는 AI가 세상을 이해하는 가장 기초적인 데이터 규격임.

# 복소수

## 1. 정의
- 정의 : 복소수 집합 C는 a + bi 형태의 순서쌍로 정의되며 이를 통해 음수의 제곱근을 다룰 수 있다. 

### 1.1 복소수의 좀 더 깊은 정의
- 보통 $x^2 = -1$ 이라는 방정식을 풀수 없다고 배웠음 
	- 실수 범위에서는 어떤 수를 제곱해도 음수가 될 수 없기 때문
	- 수학자들은 여기서 **제곱해서 -1이 되는 새로운 수i**가 있다고 정의함 
		- 실수 $\mathbb{R}$ : 1차원 직선 위의 점 (예 5, -3.2)
		- 복소수 $\mathbb{C}$ : 실수부(a)와 허수부(b)를 결합한 a + bi 형태의 수
	- 예시) : 횡스크롤 게임(1D)만 하다가 오픈 월드 게임(2D)으로 맵을 확장한 것과 같다. 이제 우리는 수직선 밖의 공간을 수식 하나로 표현할 수 있게 된것임

## 2. 복소평면 (기하학적 표현)

### 2.1 좌표계의 구성
- 실수는 왼쪽 -/ 오른쪽 +만으로 있는 1차원 직선상에 존재함. 하지만 여기서 허수 i라는 새로운 단위를 도입하면 기존의 실수 직선과 직교하는 (90도로 만나는) 새로운 축이 하나 더 생김
	- 가로축 (Real Axis) : 우리가 아는 실수 (1, 2, -5, ...)
	- 세로축 (Imaginary Axis) : 허수 단위 (i, 2i, -3i, ....)
	- 복소수 z = a + bi: 평면 위의 좌표 (a,b)에 찍히는 하나의 점

![복소평면](/assets/img/posts/linear-algebra/01_complex_plane_basic.svg)

### 2.2 기하학적 해석
- 컴퓨터 그래픽스나 로보틱스에서 복소수가 강력한 이유는 **회전** 표현을 잘함
	- 복소평면(Complex Plane): $x$축을 실수축, $y$축을 허수축으로 둠. 그러면 복소수 a+bi는 평면 위의 좌표 $(a, b)$가 됨.
- 이처럼 복소수는 크기(Magnitude)와 방향(Phase)을 동시에 저장하는 데이터 구조라고 볼수 있음

## 3. 연산

### 3.1 복소 평면에서의 연산 : 곱셈 = 회전
- 이것이 단순한 2D 벡터 좌표계와 복소평면을 가르는 결정적인 차이임. 일반적인 벡터 (x,y)끼리는 곱셈이 직관적으로 정의되지 않음. 복소수는 곱셈을 통해 공간을 변형(Transformation)할 수 있음 
	- 덧셈 : 평면 위에서의 단순한 이동 (Translation)
	- 곱셈 : 평면 위에서의 확대/축소와 회전을 동시에 수행
		- 어떤 복소수에 i를 곱하면? -> 평면 위에서 정확히 90도 회전함
		- 어떤 복소수에 -1을 곱하면? -> 180도 회전하여 반대편으로 감

![덧셈과 곱셈](/assets/img/posts/linear-algebra/06_add_vs_mul.svg)

### 3.2 i 곱셈의 구체적 의미
- i를 곱한다는 의미 : 어떤 복소수에 i를 곱하면 복소평면 위에서 시계 반대 방향으로 90도 회전 시키는 결과가 나옴
	- $1 \times  i = i$ (90도 회전)
	- $i \times i = -1$ (또 90도 회전하여 총 180도 회전)
	- $-1 \times i = -i$ (또 90도 회전하여 총 270 회전)
	- $-i \times i = 1$ (한바퀴 돌아 제자리)
- AI에서 사용 예시) LLM에서 쓰이는 RoPE 기술이 복소평면의 회전 원리를 이용함. 토큰의 위치 정보를 복소평면상의 회전 각도로 인코딩하여 모델이 단어 사이의 상대적 거리를 더 잘 파악하게 함.

![i 곱셈 회전](/assets/img/posts/linear-algebra/02_i_rotation.svg)

### 3.3 켤레와 절댓값
- 켤레 복소수 
	- 복소수 z = a+bi가 있을때 허수부의 부호만 반대로 바꾼것을 켤레 복소수라고 하며 $\bar{z}$ (또는 $z^*$)로 표기
		- 대수적 정의 $z = a + bi \implies \bar{z} = a - bi$
		- 기하적 의미 : 복소평면에서 실수축(x축)을 기준으로 대칭 이동시킨점
		- 주요 성질
			- $\overline{z + w} = \bar{z} + \bar{w}$ (덧셈에 대해 보존)
			- $\overline{z \cdot w} = \bar{z} \cdot \bar{w}$ (곱셈에 대해 보존)
			- $z + \bar{z} = 2a$ (항상 실수) : 허수 성분을 제거하여 실수부만 추출할 때 사용함
			- $z - \bar{z} = 2bi$ (순허수) : 허수 성분만 추출할 때 사용함

![켤레 복소수](/assets/img/posts/linear-algebra/03_conjugate.svg)

- 절댓값
	- 복소수의 절댓값은 복소 평면의 원점에서 해당 복소수점까지의 유클리드 거리를 의미함
		- 신호 처리에서는 진폭이나 에너지로 해석됨
	- 대수적 정의 $|z| = \sqrt{a^2 + b^2}$
	- 켤레와의 관계 $|z|^2 = z \cdot \bar{z}$
		- 이 수식은 매우 중요 복소수 자기 자신과 켤레를 곱하면 **허수부가 사라지고 크기의 제곱(실수)**가 나옴 -> 즉 모든 복소수 연산의 기초가 됨
	- 주요 성질
		- $|z \cdot w| = |z| \cdot |w|$ (곱셈의 크기는 각 크기의 곱)
		- $|\bar{z}| = |z|$ (대칭 이동해도 원점에서의 거리는 동일)
		- $|z + w| \leq |z| + |w|$ (삼각형 부등식)

![절댓값](/assets/img/posts/linear-algebra/04_modulus.svg)

- 켤레와 절댓값이 결합할때 비로소 복소수를 **복소수를 측정 가능한 양**으로 변환 가능

### 3.4 나눗셈
- 분모에 복소수가 있으면 연산이 불가능에 가깝지만, 분모의 켤레복소수를 분자/분모에 곱하면 분모가 절댓값의 제곱(실수)로 변하여 계산이 단순해짐 $$\frac{w}{z} = \frac{w \cdot \bar{z}}{z \cdot \bar{z}} = \frac{w \cdot \bar{z}}{|z|^2}$$

### 3.5 복소 벡터공간과 에르미트 내적
- 컴공에서 다루는 벡터 공간에서 두 복소수 벡터의 유사도를 구할때는 반드시 한쪽 벡터에 켤레를 취해야함
- $\langle \mathbf{u}, \mathbf{v} \rangle = \mathbf{u}^H \mathbf{v}$ (여기서 $H$는 켤레 전치/Hermitian)
- 이렇게 해야만 자기 자신과의 내적이 $|z|^2$ 형태의 양의 실수(거리)로 정의됨

## 4. 극좌표 형식 
- 복소평면 위의 점을 (a,b)라는 직교 좌표가 아니라 오일러 공식을 이용하여 표현하면 계산이 비약적으로 간단해짐 $$z = r(\cos \theta + i \sin \theta) = re^{i\theta}$$
	-  r : 원점에서의 거리 (Magnitude/Amplitude)
	-  $\theta$ : 실수축과 이루는 각도 (Phase)
- 이런 형식을 사용하면 단순한 지수 연산으로 변경됨
- 이는 신호처리(DSP)나 그래프 데이터의 주파수 분석(Spectral GNN)에서 복잡한 파동 겹침 현상을 계산할 때 연산량을 획기적으로 줄여주는 핵심 도구임

![극좌표 형식](/assets/img/posts/linear-algebra/05_polar_form.svg)

## 5. 왜 우리 분야에 중요한가?
- 단순히 수학적 유희가 아닌 실제 연구 현장에서는 다음과 같은 물리적/계산적 이점 제공
	- 신호 처리와 FFT(Fast Fourier Transform)
		- 디지털 신호나 오일러 공식을 다룰떄 복소수는 필수다. 시간 도메인의 데이터를 주파수 도메인으로 보낼 때 복소수는 진폭과 위상 정보를 한번에 캡슐화하여 연산을 매우 단순하게 함
	- 그래프 스펙트럴 이론 (Spectral Graph Theory)
		- GNN 연구에서 그래프 라플라시안 행렬을 다루다 보면 고윳값을 분석하게 됨. 이때 행렬이 비대칭인 경우 고윳값이 복소수로 나올 수 있는데, 이는 그래프 내에 방향성이 있거나 순환 구조가 있음을 시사하는 중요한 정보가 됨
	- 양자 컴퓨팅 및 최적화
		- 양자 비트의 상태는 복소수 확률 진폭으로 표현된다. 또한 특정 신경망 아키텍처에서 복소수 활성화 함수를 사용하여 파동 형태의 데이터를 더 잘 학습하기도 한다.

## 6. 간단한 정리
- 방향성이 있는 데이터를 단순히 표현이 아닌, 해당 방향을 비틀고 돌리고 합치는 연산을 편하게 할때 사용.
- 용어 정리
	- 벡터 : 두 개의 숫자를 칸막이로 나누어 담아둔 바구니와 같음. 두 바구니를 더 할 수 있음. 하지만 두 바구니를 곱해서 새로운 방향을 만드는 수학적 규칙이 직관적이지 않음
	- 복소수 : 두  숫자가 결합하여 하나의 완성된 숫자가 됨. 덕분에 실수처럼 곱하기, 나누기가 가능해짐. 이 곱셈이 기하학적으로 회전과 확대라는 효과를 가지고 있음
- 이 상황일때 무조건 복소수
	- 회전이 빈번할 때 : 행렬 곱셈 대신 복소수 곱셈 한번으로 끝낼 수 있어 계산량이 줄어듬
	- 파동이나 주기성이 있을때 : 신호의 세기(크기)와 타이밍(위상)을 하나의 숫자에 캡슐화하여 처리 할 수 있음
	- 시스템의 안정성을 분석할 때 : 제어 공학이나 그래프 이론에서 시스템이 진동하는지 발산하는지를 복소평면 위의 위치로 한눈에 파악할 수 있음
